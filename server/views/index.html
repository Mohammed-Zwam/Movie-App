/auth (Authentication)

POST
/auth/register request JSON
Register a new user. Expects JSON body with name, email, password (validated).
POST
/auth/login returns cookie
Log in an existing user. Expects JSON with email and password. Returns auth cookie on success.
GET
/auth/verify
Verify current authentication (based on cookie). Useful to check session status.
GET
/auth/logout
Log out the current user (clears auth cookie).

/api/movies (Movies)

GET
/api/movies/
Get a list of movies. Supports pagination on the server side (if implemented).
GET
/api/movies/search?query=...
Search for movies by title or other fields. Query string param: query.
GET
/api/movies/recommendations
Get recommended movies (server-side logic). Returns a curated list of suggestions.
GET
/api/movies/:id
Get detailed information for a single movie by its id.

/api/user (User-specific)

GET
/api/user/watchList auth
Get the authenticated user's watch list. Requires authentication.
POST
/api/user/watchList/:movieId auth
Toggle a movie in the user's watch list (add/remove). Requires authentication.
